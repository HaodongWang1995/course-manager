# 在线教学管理系统 PRD

### TL;DR

本项目为某教育机构定制的内部教学管理工具。支持老师创建和管理付费课程，学生浏览课程详情及课程日程。学生报名申请及审核流程将在最后迭代上线。项目采用Next.js全栈开发模式（利用API Routes与Server Components），所有数据存储于AWS Serverless架构（DynamoDB或Aurora Serverless）。适配PC和主要移动端浏览器，预期团队规模为1-2人，计划用时3-4周。

---

## Goals

### Business Goals

* 人工课程管理环节缩减80%，大幅提升教务效率。

* 系统每月运营成本≤500元人民币。

* 课程信息录入并上线用时不超过1天。

* 覆盖所有目标老师与学生的PC与手机端使用场景。

### User Goals

* 老师轻松在线发布、编辑和管理课程，包含课程内容、日程、价格与介绍。

* 学生便捷浏览机构开设的所有课程，清晰了解课程详情与安排。

* 学生报名申请与审核流程高效、透明（上线于最后迭代）。

* 用户在任何主流终端均可顺畅操作系统。

### Non-Goals

* 不涉及线上课程直播、群聊、打卡等实时互动。

* 仅适用单机构内部，不作外部公开平台使用。

* 不提供多机构账户体系与跨机构数据集成。

---

## User Stories

**老师（Teacher）**

* 作为老师，我需要在线创建课程，填写名称、简介、价格、课时日程等详细资料；

* 作为老师，我希望随时可以对既有课程做编辑、管理或下架操作；

* 作为老师，我希望后续能及时收到学生报名信息，并高效完成招生审核（Phase 4）。

**学生（Student）**

* 作为学生，我需要便捷浏览所有招生中的课程，获取完整课程详情和时间安排；

* 作为学生，我希望在后续版本中可在线申请报名，并查看我的申请进展和审核状态（Phase 4）。

---

## Functional Requirements

* 用户认证与权限管理

  * 登录/注册流程，并区分教师与学生权限；

  * 权限校验，不同角色显示对应功能；

* 课程管理（老师侧）

  * 创建新课程：编辑包括课程名称、简介、价格、日程安排等；

  * 编辑、下架课程：对现有课程进行内容修改、删除或暂时下线；

* 学生课程浏览

  * 展示全部开放中的课程，以列表和详情页方式展现；

  * 显示详细课程内容、日程与价格，支持课程筛选、搜索；

* 响应式适配

  * 页面设计自适应PC与主流手机端，交互流畅一致；

* 报名与审核流程 \[**Phase 4 上线**\]

  * 学生可在线提交报名申请，并确认基本信息；

  * 老师端实时收到报名并进行审核，通过/拒绝并自动通知结果；

  * 学生可随时查询报名状态；

* 系统提示与安全

  * 操作后给出明确反馈，异常时给出友好提示；

  * 合理限制同一账号重复报名等违规行为；

## User Experience

**入口与首次体验**

* 机构分发系统访问网址，所有用户在PC或手机端均可使用；

* 登录、注册流程清晰简捷，初次登录根据角色进入相应首页；

* 老师直达课程管理后台，学生进入课程浏览首页。

**老师端核心流程**

* 创建课程：点击“新建课程”按钮，填写标题、简介、价格及每个课时的时间安排，补充封面图片后发布；

* 管理课程：课程管理页支持对任一区课程编辑、删除、下架等操作；

* 审核报名（最后迭代）：收到学生报名通知，在审核页快速处理申请，自动推送结果反馈。

**学生端核心流程**

* 浏览课程：列表页展示所有招生中课程，支持分类、搜索，点击进入即可查看全部课程信息（不含报名功能）；

* 报名申请与状态（最后迭代）：点击“报名”填写信息后一键提交，系统实时反馈申请状态。

**UI/UX亮点**

* 完全响应式设计，所有页面在PC和主流手机端均正常显示与操作；

* 操作按钮、主要信息区域明显突出，流程简明直观，表单多步验证友好；

* 全站统一色彩主题和风格，文案提示精准，异常场景有清晰指引；

## Narrative

李老师通过系统后台创建了“Python编程入门”课程，只需输入课程基本信息、详细日程与价格，即可发布。学生在家用电脑或手机访问系统，无需注册即可浏览全部课程列表，查阅详细介绍、上课时间与费用情况。课程信息一目了然，学生可结合自己兴趣作出报名决定（报名功能Phase 4上线）。未来报名功能上线后，学生将能直接线上申请，李老师实时审核并反馈，省去传统的电话咨询与纸质流程，高效又省心。

## Technical Considerations

### 总体技术方案

* **Next.js 全栈开发**

  * 使用Next.js（最新LTS版本）开发，API Routes处理后端接口，Server Components提升渲染效率和数据安全；

  * 前后端同仓库维护，减少沟通与集成开销；

* **用户认证**

  * 推荐使用NextAuth.js实现基于邮箱/机构分配帐号的认证；

  * Session基于JWT或AWS Cognito（后期可选）实现；

* **数据存储**

  * 主表：课程（Course）、用户（User）、报名（报名相关功能Phase 4上线）；

  * 存储介质：**AWS DynamoDB**（优选，灵活Serverless）、如需关系型可用Aurora Serverless（低运维）；

  * 课程封面或图片等文件推荐用**AWS S3**对象存储；

* **API及无服务器计算**

  * Next.js API Routes处理常规请求；

  * 针对重型业务或需异步处理的任务，可适当用**AWS Lambda**；

  * 如日后需服务公开或拓展，接口可对接**AWS API Gateway**；

* **其他**

  * 系统数据加密存储与传输（HTTPS，DynamoDB安全组、KMS）

  * 定期快照与备份机制，支持一键数据导出

### 响应式设计

* 使用TailwindCSS、Ant Design for React等组件库；

* 样式按“移动优先原则”开发，全端统一体验。

### 部署与运维

* Next.js前后端代码统一部署于Vercel、AWS Amplify或Lambda@Edge等Serverless环境，无需传统运维；

* 环境变量、AWS资源权限、最小化敏感信息暴露；

* 整合CloudWatch监控、日志采集及自动告警功能。

### 安全与隐私

* 权限与视图逻辑前后端一致，防止越权访问与数据泄露；

* 全平台 HTTPS 加密传输，数据最小化获取与分级授权；

* 用户重要信息脱敏、加密存储。
