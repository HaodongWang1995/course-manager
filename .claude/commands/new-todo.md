# 新增 TODO

与用户对话，了解问题或待办事项，然后创建 TODO 文档。

## 流程

### 步骤 1 — 了解问题

询问用户：
- 问题/待办的简短标题
- 详细描述（现象、复现步骤、期望行为等）
- 涉及的页面或模块
- 优先级（如用户未指定，根据影响范围建议）

### 步骤 2 — 确认内容

整理后向用户确认：
```
【TODO 确认】
- 标题：xxx
- 优先级：P0/P1/P2
- 描述：xxx
- 涉及模块：xxx

是否准确？
```

### 步骤 3 — 创建 TODO 文件

在 `PRD/` 目录下创建文件，命名规则：`TODO-{标题简写}-{YYYYMMDD}.md`

文件模板：

```markdown
# TODO: {标题}

> 创建时间：{YYYY-MM-DD}
> 优先级：{P0 | P1 | P2}
> 状态：待修复

---

## 问题描述

{详细描述}

## 涉及模块

- {页面/文件路径}

## 复现步骤（如适用）

1. xxx
2. xxx

## 期望行为

{描述}

## 任务清单

- [ ] {子任务1}
- [ ] {子任务2}
```

**优先级定义：**
- **P0**：核心功能不可用，阻断用户流程
- **P1**：功能异常但有 workaround，或重要体验问题
- **P2**：UI 细节、优化项、非紧急改进

### 步骤 4 — 提交 git

```bash
git add PRD/
git commit -m "docs(TODO): 新增 {标题}"
```

完成后告知用户文件路径。
